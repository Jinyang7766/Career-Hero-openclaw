## 1. 信息层级（Information Hierarchy）

- [ ] 结果页首屏可见：总分/结论标签/Top 3 建议
- [ ] 解释信息位于结果下方，不抢占主决策区域

- [ ] 空态/错误态不会覆盖用户主任务入口

## 2. 文案语气（Tone & Copy）

- [ ] 成功态文案包含“已完成 + 下一步”
- [ ] 处理中文案有时间预期（如 5-15 秒）
- [ ] 降级文案明确“可继续使用 + 结果边界”
- [ ] 失败文案给出可执行动作（重试/返回/联系支持）
- [ ] 禁用词未出现：未知错误、参数非法（无解释）

验收证据：文案走查记录 `[...]`

## 3. 技术字段默认隐藏（Default Hide）

- [ ] 普通用户路径下不显示 sessionId/traceId/chunkId
- [ ] 技术详情只能通过显式操作展开（折叠区/按钮）
- [ ] 复制结果默认不包含敏感字段（token/cookie/内部 URL）
- [ ] 调试模式仅限白名单账号或预发环境启用

验收证据：操作录像 `[...]` / 配置截图 `[...]`

## 4. 异常与恢复体验

- [ ] 网络失败时可一键重试

- [ ] 检索降级时主流程可继续且有说明
- [ ] 错误提示附 requestId（仅在需要支持协同时展示）

## 6. Go / Conditional Go / No-Go（UX）

### Go（可放行）
- 第 1~4 节全部通过，且第 5 节无 P0 问题。

### Conditional Go（条件放行）
- 存在少量 P1 问题，但不影响主流程“输入 -> 分析 -> 获取建议 -> 重试恢复”。
- 已登记 Owner 与 ETA（不超过一个迭代）。

### No-Go（禁止放行）
- 结果页缺少主结论或下一步建议。
- 错误/降级场景无恢复路径。
- 技术敏感字段默认对普通用户可见。
